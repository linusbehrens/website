     76 <a href="#l57" class="line" id="l57">     57</a> &lt;p&gt;When ever editing big text projects I used &lt;span
     77 <a href="#l58" class="line" id="l58">     58</a> class=&quot;math inline&quot;&gt;$\LaTeX$&lt;/span&gt;. Just because Latex is the right
     78 <a href="#l59" class="line" id="l59">     59</a> tool for the job.&lt;/p&gt;
     79 <a href="#l60" class="line" id="l60">     60</a> &lt;p&gt;But as it turned out there has been a simpler tool for the job. And
     80 <a href="#l61" class="line" id="l61">     61</a> it is called Typst.&lt;/p&gt;
     81 <a href="#l62" class="line" id="l62">     62</a> &lt;p&gt;Why it is simpler:&lt;/p&gt;
     82 <a href="#l63" class="line" id="l63">     63</a> &lt;ul&gt;
     83 <a href="#l64" class="line" id="l64">     64</a> &lt;li&gt;It is FOSS&lt;/li&gt;
     84 <a href="#l65" class="line" id="l65">     65</a> &lt;li&gt;Ready to write: no &lt;code&gt;\document{begin}&lt;/code&gt;&lt;/li&gt;
     85 <a href="#l66" class="line" id="l66">     66</a> &lt;li&gt;Markdown based syntax where it can: eg. headings with
     86 <a href="#l67" class="line" id="l67">     67</a> &lt;code&gt;=&lt;/code&gt;&lt;/li&gt;
     87 <a href="#l68" class="line" id="l68">     68</a> &lt;li&gt;Easy math syntax&lt;/li&gt;
     88 <a href="#l69" class="line" id="l69">     69</a> &lt;li&gt;Simple coding constructs instead of single use case based
     89 <a href="#l70" class="line" id="l70">     70</a> macros&lt;/li&gt;
     90 <a href="#l71" class="line" id="l71">     71</a> &lt;li&gt;FOSS templates ready to clone&lt;/li&gt;
     91 <a href="#l72" class="line" id="l72">     72</a> &lt;li&gt;Renders preview from buffer: instant update without having to
     92 <a href="#l73" class="line" id="l73">     73</a> safe&lt;/li&gt;
