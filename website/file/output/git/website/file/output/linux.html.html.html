     76 <a href="#l57" class="line" id="l57">     57</a> &lt;h1 id=&quot;why-gnulinux&quot;&gt;Why GNU/Linux&lt;/h1&gt;
     77 <a href="#l58" class="line" id="l58">     58</a> &lt;blockquote&gt;
     78 <a href="#l59" class="line" id="l59">     59</a> &lt;p&gt;This is the Unix philosophy: Write programs that do one thing and do
     79 <a href="#l60" class="line" id="l60">     60</a> it well. Write programs to work together. Write programs to handle text
     80 <a href="#l61" class="line" id="l61">     61</a> streams, because that is a univeRsal interface&lt;/p&gt;
     81 <a href="#l62" class="line" id="l62">     62</a> &lt;p&gt;&lt;em&gt;- Doug McIlroy, the inventor of Unix pipes and one of the
     82 <a href="#l63" class="line" id="l63">     63</a> founders of the Unix tradition&lt;/em&gt;&lt;/p&gt;
     83 <a href="#l64" class="line" id="l64">     64</a> &lt;/blockquote&gt;
     84 <a href="#l65" class="line" id="l65">     65</a> &lt;p&gt;Linux at its core is the idea of intercompatability born in the body
     85 <a href="#l66" class="line" id="l66">     66</a> of a kernel. What I love in Linux is not how fast and free it is. Not
     86 <a href="#l67" class="line" id="l67">     67</a> how amazingly many people contribute to one goal. It is the way I use it
     87 <a href="#l68" class="line" id="l68">     68</a> and how it allowed me to fit the most bare bones software to my
     88 <a href="#l69" class="line" id="l69">     69</a> needs.&lt;/p&gt;
     89 <a href="#l70" class="line" id="l70">     70</a> &lt;h2 id=&quot;examples-for-the-perfect-fit&quot;&gt;Examples for the perfect fit&lt;/h2&gt;
     90 <a href="#l71" class="line" id="l71">     71</a> &lt;p&gt;My senior high note taking setup is pretty complex. However it does
     91 <a href="#l72" class="line" id="l72">     72</a> fit my workflow perfectly.&lt;/p&gt;
     92 <a href="#l73" class="line" id="l73">     73</a> &lt;ul&gt;
     93 <a href="#l74" class="line" id="l74">     74</a> &lt;li&gt;&lt;a href=&quot;/posts/neovim&quot;&gt;neovim&lt;/a&gt; note taking&lt;/li&gt;
     94 <a href="#l75" class="line" id="l75">     75</a> &lt;li&gt;&lt;a href=&quot;posts/tex&quot;&gt;&lt;/a&gt; for long-term sheets&lt;/li&gt;
     95 <a href="#l76" class="line" id="l76">     76</a> &lt;li&gt;&lt;a href=&quot;posts/markdown&quot;&gt;markdown&lt;/a&gt; for short-term notes in
     96 <a href="#l77" class="line" id="l77">     77</a> lessons&lt;/li&gt;
     97 <a href="#l78" class="line" id="l78">     78</a> &lt;li&gt;pandoc turns markdown into &lt;code&gt;\tex&lt;/code&gt;&lt;/li&gt;
     98 <a href="#l79" class="line" id="l79">     79</a> &lt;li&gt;telescope and todo-comments for todo lists&lt;/li&gt;
     99 <a href="#l80" class="line" id="l80">     80</a> &lt;li&gt;&lt;a href=&quot;posts/anki&quot;&gt;anki&lt;/a&gt; for everyday learning&lt;/li&gt;
    100 <a href="#l81" class="line" id="l81">     81</a> &lt;li&gt;github for backing up and syncing my data (homelab in the
    101 <a href="#l82" class="line" id="l82">     82</a> future)&lt;/li&gt;
    102 <a href="#l83" class="line" id="l83">     83</a> &lt;/ul&gt;
    103 <a href="#l84" class="line" id="l84">     84</a> &lt;p&gt;With that setup I am albe to quickly make notes in my lessons as well
    104 <a href="#l85" class="line" id="l85">     85</a> as clearly structure my long-term sheets. With having anki on any of my
    105 <a href="#l86" class="line" id="l86">     86</a> devices I am able to learn everywhere.&lt;/p&gt;
    106 <a href="#l87" class="line" id="l87">     87</a> &lt;p&gt;If I want I could even run all of that over ssh from a homelab of
    107 <a href="#l88" class="line" id="l88">     88</a> server at school.&lt;/p&gt;
    108 <a href="#l89" class="line" id="l89">     89</a> &lt;h2 id=&quot;intercompatability&quot;&gt;Intercompatability&lt;/h2&gt;
    109 <a href="#l90" class="line" id="l90">     90</a> &lt;p&gt;My shell runs in &lt;a
    110 <a href="#l91" class="line" id="l91">     91</a> href=&quot;https://github.com/alacritty/alacritty&quot;&gt;alacritty&lt;/a&gt; (macOS) and
    111 <a href="#l92" class="line" id="l92">     92</a> &lt;a href=&quot;https://st.suckless.org/&quot;&gt;st&lt;/a&gt; (arch and void). My most used
    112 <a href="#l93" class="line" id="l93">     93</a> and centered tool is &lt;a href=&quot;/posts/neovim&quot;&gt;neovim&lt;/a&gt;. With my
    113 <a href="#l94" class="line" id="l94">     94</a> configuration it is beautiful and works seemlesly. With telescope I
    114 <a href="#l95" class="line" id="l95">     95</a> quickly open my &lt;code&gt;&amp;lt;subject&amp;gt;.refile.md&lt;/code&gt; note and later
    115 <a href="#l96" class="line" id="l96">     96</a> the week I convert it to &lt;code&gt;&amp;lt;subject.refile.tex&lt;/code&gt; with
    116 <a href="#l97" class="line" id="l97">     97</a> pandoc. The important information gets refiled into
    117 <a href="#l98" class="line" id="l98">     98</a> &lt;code&gt;&amp;lt;subject&amp;gt;.tex&lt;/code&gt; and tagged as
    118 <a href="#l99" class="line" id="l99">     99</a> &lt;code&gt;ANKI: &amp;lt;subject&amp;gt;&lt;/code&gt;. So if I open telescope to see my
    119 <a href="#l100" class="line" id="l100">    100</a> todos the anki sign reminds me of entering the information to my
    120 <a href="#l101" class="line" id="l101">    101</a> &lt;code&gt;anki deck&lt;/code&gt;. Every evening I push my files to github to be
    121 <a href="#l102" class="line" id="l102">    102</a> safe for a data loss.&lt;/p&gt;
    122 <a href="#l103" class="line" id="l103">    103</a> &lt;h2 id=&quot;options-example-distributions&quot;&gt;Options (example
    123 <a href="#l104" class="line" id="l104">    104</a> distributions)&lt;/h2&gt;
    124 <a href="#l105" class="line" id="l105">    105</a> &lt;p&gt;The following are distros I have used.&lt;/p&gt;
    125 <a href="#l106" class="line" id="l106">    106</a> &lt;ul&gt;
    126 <a href="#l107" class="line" id="l107">    107</a> &lt;li&gt;Manjaro (Why does it exists exactly)&lt;/li&gt;
    127 <a href="#l108" class="line" id="l108">    108</a> &lt;li&gt;Mint (If you would want to cheat)&lt;/li&gt;
    128 <a href="#l109" class="line" id="l109">    109</a> &lt;li&gt;&lt;a href=&quot;/content/arch.md&quot;&gt;Arch&lt;/a&gt; (Amazing documentation, pacman,
    129 <a href="#l110" class="line" id="l110">    110</a> AUR)&lt;/li&gt;
    130 <a href="#l111" class="line" id="l111">    111</a> &lt;li&gt;Arco (No systemd arch)&lt;/li&gt;
    131 <a href="#l112" class="line" id="l112">    112</a> &lt;li&gt;Gentoo (If you want it you know it)&lt;/li&gt;
    132 <a href="#l113" class="line" id="l113">    113</a> &lt;li&gt;Void (small, but no systemd!)&lt;/li&gt;
    133 <a href="#l114" class="line" id="l114">    114</a> &lt;/ul&gt;
    134 <a href="#l115" class="line" id="l115">    115</a> &lt;p&gt;On my &lt;strong&gt;x61s ThinkPad&lt;/strong&gt; I run Arch Linux. It is light
    135 <a href="#l116" class="line" id="l116">    116</a> weight and has amazing documentary. The arch wiki is one of the best
    136 <a href="#l117" class="line" id="l117">    117</a> documents and I even refer to it to non specific arch users.&lt;/p&gt;
    137 <a href="#l118" class="line" id="l118">    118</a> &lt;p&gt;The &lt;strong&gt;2013 Mac mini&lt;/strong&gt; runs void linux. Not because I
    138 <a href="#l119" class="line" id="l119">    119</a> don’t like arch as much, but because after inserting a bootable usb and
    139 <a href="#l120" class="line" id="l120">    120</a> entering grubs “install arch for x86_64” I got a black screen for days.
    140 <a href="#l121" class="line" id="l121">    121</a> I choose void because I liked the neofetch batch and because it has no
    141 <a href="#l122" class="line" id="l122">    122</a> &lt;code&gt;systemd&lt;/code&gt; elevating the performence and reducing drama. I
    142 <a href="#l123" class="line" id="l123">    123</a> mainly use that mashine as a ssh server.&lt;/p&gt;
    143 <a href="#l124" class="line" id="l124">    124</a> &lt;p&gt;Both mashines run &lt;a href=&quot;https://www.suckless.org&quot;&gt;suckless
    144 <a href="#l125" class="line" id="l125">    125</a> software&lt;/a&gt;. Namely &lt;code&gt;dwm&lt;/code&gt;, &lt;code&gt;dmenu&lt;/code&gt;, and
    145 <a href="#l126" class="line" id="l126">    126</a> &lt;code&gt;st&lt;/code&gt;. I am really happy with how good it works, but I do not
    146 <a href="#l127" class="line" id="l127">    127</a> feel well using &lt;code&gt;X&lt;/code&gt;.&lt;/p&gt;
    147 <a href="#l128" class="line" id="l128">    128</a> &lt;p&gt;My &lt;strong&gt;MacBook&lt;/strong&gt; runs macOS because Asahi sucks at the
    148 <a href="#l129" class="line" id="l129">    129</a> moment. However I use the tiling window manager &lt;code&gt;yabaii&lt;/code&gt; and
    149 <a href="#l130" class="line" id="l130">    130</a> &lt;a href=&quot;https://github.com/alacritty/alacritty&quot;&gt;alacritty&lt;/a&gt; to come
    150 <a href="#l131" class="line" id="l131">    131</a> closer to a linux experience.&lt;/p&gt;
